Setup Your GitHub Repository

1

Create a GitHub account

If you don't already have a GitHub account, sign up for free.

2

Create a new repository

Name your repository restaurant-menu or any name you prefer. Make sure the repository is set to Public.

Tip: For a professional URL, name your repository as menu if your GitHub username is your restaurant name.
3

Enable GitHub Pages

Go to your repository settings, scroll down to "GitHub Pages" section and select:

  • Source: main branch
  • Folder: / (root)
  • Click Save

GitHub will provide you with a URL like: https://yourusername.github.io/restaurant-menu/

4

Generate a Personal Access Token

This will allow our tool to upload files to your repository:

  1. Go to GitHub Settings ‚Üí Developer settings ‚Üí Personal access tokens
  2. Click Generate new token
  3. Name it "Menu QR Upload"
  4. Select the repo scope
  5. Click Generate token
  6. Copy your token and save it somewhere safe
Important: You will only see this token once, so make sure to copy it!

This token is only used to upload files and is never stored on our servers.

Frequently Asked Questions

Everything you need to know about our QR menu solution

Is this really completely free?

Yes! GitHub Pages provides free hosting for public repositories with unlimited bandwidth. There are no hidden costs or premium features.

How do I update my menu?

Simply upload a new version of your PDF with the same name. GitHub will automatically replace the old version, and your QR code will continue to work without any changes.

Is there a limit to how many menus I can upload?

GitHub has a soft limit of 1GB per repository, which is more than enough for thousands of menu PDFs. You can also create multiple repositories if needed.

How secure is GitHub Pages for hosting my menus?

GitHub Pages is extremely reliable and secure, used by major companies and organizations worldwide. Your menu will be served over HTTPS for added security.

Can I use a custom domain name?

Yes! GitHub Pages allows you to use your own custom domain. In your repository settings, you can configure a custom domain like "menu.yourrestaurant.com".
